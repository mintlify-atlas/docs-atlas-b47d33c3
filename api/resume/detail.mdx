## Get Resume Detail

Fetches detailed information about a specific resume, including complete analysis history with all scores and suggestions, parsed text content, and associated interview sessions.

### Endpoint

```
GET /api/resumes/{id}/detail
```

### Authentication

This endpoint requires authentication. Include your authentication token in the request headers.

### Path Parameters

<ParamField path="id" type="long" required>
  The unique identifier of the resume to retrieve
</ParamField>

### Response

<ResponseField name="code" type="integer">
  Status code. `200` indicates success.
</ResponseField>

<ResponseField name="message" type="string">
  Response message
</ResponseField>

<ResponseField name="data" type="object">
  Detailed resume information
  
  <Expandable title="Resume detail properties">
    <ResponseField name="id" type="long">
      Unique identifier for the resume
    </ResponseField>
    
    <ResponseField name="filename" type="string">
      Original filename of the uploaded resume
    </ResponseField>
    
    <ResponseField name="fileSize" type="long">
      Size of the resume file in bytes
    </ResponseField>
    
    <ResponseField name="contentType" type="string">
      MIME type of the uploaded file (e.g., `application/pdf`, `application/vnd.openxmlformats-officedocument.wordprocessingml.document`)
    </ResponseField>
    
    <ResponseField name="storageUrl" type="string">
      Direct URL to access the stored resume file
    </ResponseField>
    
    <ResponseField name="uploadedAt" type="string">
      ISO 8601 timestamp when the resume was uploaded
    </ResponseField>
    
    <ResponseField name="accessCount" type="integer">
      Number of times the resume has been accessed
    </ResponseField>
    
    <ResponseField name="resumeText" type="string">
      Parsed text content extracted from the resume file
    </ResponseField>
    
    <ResponseField name="analyzeStatus" type="string">
      Current analysis status. Values: `PENDING`, `PROCESSING`, `COMPLETED`, `FAILED`
    </ResponseField>
    
    <ResponseField name="analyzeError" type="string" optional>
      Error message if analysis failed. `null` if no error.
    </ResponseField>
    
    <ResponseField name="analyses" type="array">
      Array of all analysis records for this resume (newest first)
      
      <Expandable title="Analysis history item properties">
        <ResponseField name="id" type="long">
          Unique identifier for this analysis record
        </ResponseField>
        
        <ResponseField name="overallScore" type="integer">
          Overall resume quality score (0-100)
        </ResponseField>
        
        <ResponseField name="contentScore" type="integer">
          Content quality and relevance score (0-100)
        </ResponseField>
        
        <ResponseField name="structureScore" type="integer">
          Resume structure and organization score (0-100)
        </ResponseField>
        
        <ResponseField name="skillMatchScore" type="integer">
          Skill relevance and keyword matching score (0-100)
        </ResponseField>
        
        <ResponseField name="expressionScore" type="integer">
          Language clarity and expression score (0-100)
        </ResponseField>
        
        <ResponseField name="projectScore" type="integer">
          Project experience quality score (0-100)
        </ResponseField>
        
        <ResponseField name="summary" type="string">
          AI-generated summary of the resume analysis
        </ResponseField>
        
        <ResponseField name="analyzedAt" type="string">
          ISO 8601 timestamp when this analysis was completed
        </ResponseField>
        
        <ResponseField name="strengths" type="array">
          List of identified strengths (array of strings)
        </ResponseField>
        
        <ResponseField name="suggestions" type="array">
          List of improvement suggestions (array of objects with `category` and `description`)
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="interviews" type="array">
      Array of interview sessions associated with this resume
      
      <Expandable title="Interview history item properties">
        <ResponseField name="sessionId" type="string">
          Unique identifier for the interview session
        </ResponseField>
        
        <ResponseField name="createdAt" type="string">
          ISO 8601 timestamp when the interview was created
        </ResponseField>
        
        <ResponseField name="status" type="string">
          Interview session status (e.g., `IN_PROGRESS`, `COMPLETED`)
        </ResponseField>
        
        <ResponseField name="questionCount" type="integer">
          Number of questions in the interview
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

### Examples

<CodeGroup>

```bash cURL
curl -X GET https://api.example.com/api/resumes/12345/detail \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.example.com/api/resumes/12345/detail', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});

const result = await response.json();
console.log(result.data);
```

```python Python
import requests

url = "https://api.example.com/api/resumes/12345/detail"
headers = {"Authorization": "Bearer YOUR_TOKEN"}

response = requests.get(url, headers=headers)
print(response.json())
```

### Understanding Analysis Scores

<AccordionGroup>
  <Accordion title="Overall Score (overallScore)">
    Weighted average of all individual scores, representing the overall resume quality:
    
    - **90-100**: Exceptional - Ready for senior positions
    - **80-89**: Excellent - Competitive for most roles
    - **70-79**: Good - Solid foundation with room for improvement
    - **60-69**: Average - Several areas need enhancement
    - **Below 60**: Needs significant revision
  </Accordion>
  
  <Accordion title="Content Score (contentScore)">
    Evaluates the quality and relevance of resume content:
    
    - Relevance of work experience
    - Quality of project descriptions
    - Clarity of achievements
    - Professional accomplishments
  </Accordion>
  
  <Accordion title="Structure Score (structureScore)">
    Assesses the organization and formatting:
    
    - Logical section arrangement
    - Consistent formatting
    - Visual hierarchy
    - Readability and white space usage
  </Accordion>
  
  <Accordion title="Skill Match Score (skillMatchScore)">
    Measures skill relevance and keyword optimization:
    
    - Technical skills coverage
    - Industry-relevant keywords
    - Tool and technology mentions
    - Certifications and qualifications
  </Accordion>
  
  <Accordion title="Expression Score (expressionScore)">
    Evaluates language quality and communication:
    
    - Grammar and spelling
    - Professional tone
    - Concise expression
    - Action verb usage
  </Accordion>
  
  <Accordion title="Project Score (projectScore)">
    Assesses the quality of project experience descriptions:
    
    - Project complexity and scale
    - Role and responsibilities clarity
    - Measurable outcomes
    - Technical challenges addressed
  </Accordion>
</AccordionGroup>

### Polling for Analysis Completion

<Info>
When a resume is uploaded, analysis runs asynchronously. Implement polling to check completion:

```javascript
async function pollForAnalysis(resumeId) {
  const maxAttempts = 30;
  const interval = 2000; // 2 seconds
  
  for (let i = 0; i < maxAttempts; i++) {
    const response = await fetch(`/api/resumes/${resumeId}/detail`);
    const data = await response.json();
    
    if (data.data.analyzeStatus === 'COMPLETED') {
      return data.data;
    }
    
    if (data.data.analyzeStatus === 'FAILED') {
      throw new Error(data.data.analyzeError);
    }
    
    await new Promise(resolve => setTimeout(resolve, interval));
  }
  
  throw new Error('Analysis timeout');
}
```
</Info>
