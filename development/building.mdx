This guide covers how to build, run, and test the InterviewGuide project for local development.

## Prerequisites

Before building the project, ensure you have the following installed:

<CardGroup cols={2}>
  <Card title="Backend Requirements" icon="java">
    - **Java 21** (OpenJDK or Oracle JDK)
    - **Gradle 8.8+** (included via wrapper)
    - **PostgreSQL 14+** with pgvector extension
    - **Redis 6+**
  </Card>
  <Card title="Frontend Requirements" icon="react">
    - **Node.js 18+**
    - **pnpm 10.26.2** (specified in package.json)
    - Modern web browser
  </Card>
</CardGroup>

<Info>
  The project uses **Gradle Wrapper** for backend and **pnpm** for frontend, so you don't need to install these tools globally. The wrapper scripts will download the correct versions automatically.
</Info>

## Backend Development

### Project Structure

The backend is a **Gradle multi-project build**:

```
interviewguide/
├── app/                    # Main Spring Boot application
│   ├── src/
│   └── build.gradle
├── gradle/                 # Gradle wrapper and dependency management
│   └── libs.versions.toml # Centralized version catalog
├── gradlew                # Unix wrapper script
├── gradlew.bat            # Windows wrapper script
└── settings.gradle        # Project configuration
```

### Gradle Commands

<Tabs>
  <Tab title="Development">
    **Run the application in development mode**:

    ```bash
    ./gradlew bootRun
    ```

    This starts the Spring Boot application on `http://localhost:8080`.

    **Run with specific profile**:

    ```bash
    ./gradlew bootRun --args='--spring.profiles.active=dev'
    ```

    **Run with custom port**:

    ```bash
    ./gradlew bootRun --args='--server.port=9090'
    ```

    <Note>
      Hot reload is enabled via Spring Boot DevTools when running in development mode. Changes to Java files will trigger automatic restart.
    </Note>
  </Tab>

  <Tab title="Building">
    **Build the project**:

    ```bash
    ./gradlew build
    ```

    This compiles the code, runs tests, and creates executable JAR files in `app/build/libs/`.

    **Build without tests**:

    ```bash
    ./gradlew build -x test
    ```

    **Clean build** (removes previous build artifacts):

    ```bash
    ./gradlew clean build
    ```

    **Build output**:
    ```
    app/build/libs/
    ├── app-0.0.1-SNAPSHOT.jar              # Executable JAR
    └── app-0.0.1-SNAPSHOT-plain.jar        # Non-executable JAR
    ```

    **Run the built JAR**:

    ```bash
    java -jar app/build/libs/app-0.0.1-SNAPSHOT.jar
    ```
  </Tab>

  <Tab title="Testing">
    **Run all tests**:

    ```bash
    ./gradlew test
    ```

    **Run tests for specific module**:

    ```bash
    ./gradlew :app:test
    ```

    **Run specific test class**:

    ```bash
    ./gradlew test --tests ResumeUploadServiceTest
    ```

    **Run tests with coverage**:

    ```bash
    ./gradlew test jacocoTestReport
    ```

    **Test output locations**:
    - Test reports: `app/build/reports/tests/test/index.html`
    - Coverage reports: `app/build/reports/jacoco/test/html/index.html`

    <Info>
      Currently, the project has minimal test coverage. Contributions to improve test coverage are welcome!
    </Info>
  </Tab>

  <Tab title="Other Tasks">
    **View all available tasks**:

    ```bash
    ./gradlew tasks
    ```

    **View dependency tree**:

    ```bash
    ./gradlew dependencies
    ```

    **Check for dependency updates**:

    ```bash
    ./gradlew dependencyUpdates
    ```

    **Generate Javadoc**:

    ```bash
    ./gradlew javadoc
    ```

    Output: `app/build/docs/javadoc/index.html`

    **Format code** (if spotless plugin is configured):

    ```bash
    ./gradlew spotlessApply
    ```
  </Tab>
</Tabs>

### Build Configuration

<Accordion title="build.gradle">
  The main build configuration file:

  ```gradle
  plugins {
      id 'java'
      alias(libs.plugins.spring.boot)
      alias(libs.plugins.spring.dependency.management)
  }

  group = 'com.interview'
  version = '0.0.1-SNAPSHOT'

  java {
      toolchain {
          languageVersion = JavaLanguageVersion.of(21)
      }
  }

  dependencies {
      // Spring Boot Starters
      implementation 'org.springframework.boot:spring-boot-starter-webmvc'
      implementation 'org.springframework.boot:spring-boot-starter-data-jpa'
      implementation 'org.springframework.boot:spring-boot-starter-validation'

      // Spring AI 2.0
      implementation "org.springframework.ai:spring-ai-starter-model-openai:${libs.versions.spring.ai.get()}"
      implementation "org.springframework.ai:spring-ai-starter-vector-store-pgvector:${libs.versions.spring.ai.get()}"

      // Document parsing
      implementation libs.tika.core
      implementation libs.tika.parsers

      // Storage
      implementation "software.amazon.awssdk:s3:${libs.versions.aws.sdk.get()}"

      // Async processing
      implementation "org.redisson:redisson-spring-boot-starter:${libs.versions.redisson.get()}"

      // PDF export
      implementation "com.itextpdf:itext-core:${libs.versions.itext.get()}"
      implementation "com.itextpdf:font-asian:${libs.versions.itext.get()}"

      // Object mapping
      implementation "org.mapstruct:mapstruct:${libs.versions.mapstruct.get()}"
      annotationProcessor "org.mapstruct:mapstruct-processor:${libs.versions.mapstruct.get()}"

      // Lombok (must come before MapStruct processor)
      compileOnly libs.lombok
      annotationProcessor libs.lombok
      annotationProcessor 'org.projectlombok:lombok-mapstruct-binding:0.2.0'

      // Database
      runtimeOnly 'org.postgresql:postgresql'

      // Testing
      testImplementation 'org.springframework.boot:spring-boot-starter-test'
      testImplementation libs.junit.jupiter
  }

  tasks.named('test') {
      useJUnitPlatform()
  }
  ```
</Accordion>

### Environment Configuration

Create an `application.yml` or `application.properties` file in `app/src/main/resources/` (or use environment variables):

<CodeGroup>
```yaml application.yml
spring:
  application:
    name: interview-guide
  
  datasource:
    url: jdbc:postgresql://localhost:5432/interview_guide
    username: postgres
    password: your_password
  
  jpa:
    hibernate:
      ddl-auto: update
    show-sql: false
  
  ai:
    openai:
      api-key: ${DASHSCOPE_API_KEY}
      base-url: https://dashscope.aliyuncs.com/compatible-mode/v1
      chat:
        options:
          model: qwen-plus
  
  data:
    redis:
      host: localhost
      port: 6379

app:
  storage:
    type: s3
    s3:
      endpoint: http://localhost:8333
      bucket: interview-guide
      access-key: ${S3_ACCESS_KEY}
      secret-key: ${S3_SECRET_KEY}
  
  file:
    allowed-types:
      - application/pdf
      - application/vnd.openxmlformats-officedocument.wordprocessingml.document
      - application/msword
      - text/plain
      - text/markdown
```

```properties application.properties
spring.application.name=interview-guide

# Database
spring.datasource.url=jdbc:postgresql://localhost:5432/interview_guide
spring.datasource.username=postgres
spring.datasource.password=your_password
spring.jpa.hibernate.ddl-auto=update

# AI Configuration
spring.ai.openai.api-key=${DASHSCOPE_API_KEY}
spring.ai.openai.base-url=https://dashscope.aliyuncs.com/compatible-mode/v1
spring.ai.openai.chat.options.model=qwen-plus

# Redis
spring.data.redis.host=localhost
spring.data.redis.port=6379

# Storage
app.storage.type=s3
app.storage.s3.endpoint=http://localhost:8333
app.storage.s3.bucket=interview-guide
app.storage.s3.access-key=${S3_ACCESS_KEY}
app.storage.s3.secret-key=${S3_SECRET_KEY}
```
</CodeGroup>

<Tip>
  Use environment variables for sensitive values like API keys. Copy `.env.example` to `.env` and populate with your values.
</Tip>

### pnpm Commands

<Tabs>
  <Tab title="Development">
    **Install dependencies**:

    ```bash
    cd frontend
    pnpm install
    ```

    **Start development server**:

    ```bash
    pnpm dev
    ```

    This starts Vite dev server on `http://localhost:5173` with hot module replacement (HMR).

    **Development with custom port**:

    ```bash
    pnpm dev --port 3000
    ```

    **Development with host exposed**:

    ```bash
    pnpm dev --host
    ```

    This allows access from other devices on your network.
  </Tab>

  <Tab title="Building">
    **Build for production**:

    ```bash
    pnpm build
    ```

    This:
    1. Runs TypeScript compiler (`tsc`) to check types
    2. Builds optimized production bundle with Vite
    3. Outputs to `frontend/dist/`

    **Build output**:
    ```
    frontend/dist/
    ├── index.html
    ├── assets/
    │   ├── index-[hash].js      # Main bundle
    │   ├── index-[hash].css     # Styles
    │   └── [chunks]-[hash].js   # Code-split chunks
    └── favicon.ico
    ```

    **Preview production build**:

    ```bash
    pnpm preview
    ```

    Starts a local server to preview the production build on `http://localhost:4173`.
  </Tab>

  <Tab title="Type Checking">
    **Run TypeScript type checking**:

    ```bash
    pnpm tsc
    ```

    Or directly:

    ```bash
    npx tsc --noEmit
    ```

    **Watch mode**:

    ```bash
    npx tsc --noEmit --watch
    ```

    <Info>
      Type checking is automatically run during `pnpm build`. No need to run it separately unless you want to check types without building.
    </Info>
  </Tab>

  <Tab title="Linting">
    <Warning>
      The project currently does not have ESLint configured. It's recommended to add ESLint for code quality.
    </Warning>

    **To add ESLint** (optional):

    ```bash
    pnpm add -D eslint @typescript-eslint/eslint-plugin @typescript-eslint/parser
    pnpm add -D eslint-plugin-react eslint-plugin-react-hooks
    ```

    Then create `.eslintrc.json` and run:

    ```bash
    pnpm eslint src/
    ```
  </Tab>
</Tabs>

## Docker Deployment

For production deployment, use Docker Compose:

```bash
# Build and start all services
docker-compose up -d

# View logs
docker-compose logs -f

# Stop all services
docker-compose down

# Rebuild specific service
docker-compose up -d --build app
```

See [Docker Deployment](/deployment/docker) for detailed instructions.

## Common Development Workflows

### Full Stack Development

**Terminal 1 - Backend**:
```bash
./gradlew bootRun
```

**Terminal 2 - Frontend**:
```bash
cd frontend
pnpm dev
```

**Terminal 3 - Infrastructure** (if running locally):
```bash
# Start PostgreSQL
docker run -d --name postgres -p 5432:5432 -e POSTGRES_PASSWORD=postgres pgvector/pgvector:pg14

# Start Redis
docker run -d --name redis -p 6379:6379 redis:latest

# Start MinIO (S3-compatible storage)
docker run -d --name minio -p 9000:9000 -p 9001:9001 minio/minio server /data --console-address ":9001"
```

### Hot Reload

- **Backend**: Spring Boot DevTools automatically restarts the application when you change Java files
- **Frontend**: Vite HMR updates the browser instantly when you change React components

## Troubleshooting

<AccordionGroup>
  <Accordion title="Gradle build fails with 'Could not resolve dependencies'">
    **Solution**:
    1. Check your internet connection
    2. Clear Gradle cache: `rm -rf ~/.gradle/caches`
    3. Try running with `--refresh-dependencies`: `./gradlew build --refresh-dependencies`
    4. If using Aliyun mirrors, ensure they are accessible
  </Accordion>

  <Accordion title="Spring Boot fails to start with 'Cannot load driver class'">
    **Solution**:
    1. Ensure PostgreSQL is running
    2. Check database connection settings in `application.yml`
    3. Verify `org.postgresql:postgresql` is in dependencies
  </Accordion>

  <Accordion title="Frontend build fails with 'Cannot find module'">
    **Solution**:
    1. Delete `node_modules` and `pnpm-lock.yaml`
    2. Run `pnpm install` again
    3. Ensure you're using the correct pnpm version: `pnpm --version`
  </Accordion>

  <Accordion title="CORS errors when calling backend from frontend">
    **Solution**:
    1. Ensure `CorsConfig` is configured correctly in backend
    2. Check that frontend is using correct API base URL
    3. Verify backend is running and accessible
  </Accordion>

  <Accordion title="MapStruct generates outdated code">
    **Solution**:
    1. Clean build: `./gradlew clean`
    2. Rebuild: `./gradlew build`
    3. Ensure annotation processing is enabled in your IDE
    4. In IntelliJ IDEA: Settings → Build, Execution, Deployment → Compiler → Annotation Processors → Enable annotation processing
  </Accordion>
</AccordionGroup>
